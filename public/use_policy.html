Use Policy for GitLab and Git services provided by git.unl.edu

What this system provides

UNL Office of Technology Services (ITS) operates the git.unl.edu server running GitLab source code management and collaboration software for use by UNL staff, students, faculty, and associates. This service is intended to provide a convenient place to collaborate on software development projects at UNL. The Gitlab software provides source code management primarily in the form of Git revision control. It also provides a web interface containing additional project collaboration tools for managing development teams and software issues.

Users of the system

The git.unl.edu system is available to all UNL faculty, students, and staff having a valid UNL login. Users of the git.unl.edu system include these individuals as well as those using public key authentication for Git repository access via SSH.

Restrictions on use

All users of the git.unl.edu system are expected to adhere to "Guidelines for Use of Computing Resources at UNL", including "Executive Memorandum No. 16". Users of and projects on the system are subject to review for compliance with these policies.

Service Limitations

While ITS provides the git.unl.edu system as a resource for all UNL software development efforts, there are space and performance limitations on the server. Periodically ITS will review or audit usage of the system to ensure acceptable performance. ITS will strive to update resources available to facilitate acceptable system performance. If necessary, restrictions may be imposed on disk usage and access. If restrictions are necessary, users will be informed in as timely manner as possible.

As with all computing systems provided by ITS, git.unl.edu may be subject to occasional downtime for maintenence or repairs. ITS will strive to keep downtime minimal and will communicate any known issues through IS-Alert.

This use policy is subject to change at the discretion of ITS.
